# Status: Binder sidecar smoke test on LG webOS

Date: 2026-05-16

Target:

- Device: LG webOS TV
- Kernel: 4.4.84-229.1.kavir.2
- Arch: aarch64
- Binder protocol: 8

Confirmed working:

- binder.ko loads as external module
- /dev/binder appears as misc device major 10 minor 53
- BINDER_VERSION works
- BINDER_SET_MAX_THREADS works
- BINDER_SET_CONTEXT_MGR works
- mmap allocation shim works
- BC_TRANSACTION / BR_TRANSACTION works
- BR_REPLY works
- BINDER_TYPE_BINDER -> BINDER_TYPE_HANDLE object passing works
- BC_ACQUIRE / BR_ACQUIRE works
- BC_INCREFS_DONE / BC_ACQUIRE_DONE works
- BC_REQUEST_DEATH_NOTIFICATION works
- mini service manager works
- addService works
- getService works
- service handle ping validation works
- echo client/service smoke test works

Known limitation:

- Only /dev/binder appears on this target.
- /dev/hwbinder and /dev/vndbinder are not currently created.
- Module is permanent until reboot.
- Current mmap shim is experimental and verbose.
